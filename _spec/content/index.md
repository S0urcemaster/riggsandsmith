# Content

This section holds concrete in-game text material.

The client language is English.

In-game text should be written in English.

Terminal build output is also client-facing product text for this project.

Terminal commands, prompts, action labels, warnings, summaries, and result text should therefore be written in English.

The first product direction does not include localization.

Content entries should therefore define the English client text directly rather than storing German source text for later translation.

Content files should be agent-operable source material rather than loose notes.

Each content entry should define:

- status
- use
- context
- text
- intent
- tone
- variables, if any

## Status Values

`draft` means the text expresses a possible direction but may still change freely.

`accepted` means the text should be treated as current product content until explicitly replaced.

`placeholder` means the entry marks a needed text slot without final wording.

## Content Rules

In-game text should support the product's systems, tone, and readable consequence structure.

Text should make state changes understandable without turning the interface into exposition only.

Text should prefer short, concrete, atmospheric phrasing over long explanation.

System text may be functional and direct when the player needs clarity.

Fictional flavor text may be more suggestive, but it should still help the player read the current situation.
